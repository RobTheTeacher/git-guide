Git Status

git status […​] [--] […​]

Shows files that have changed. Displays paths that have differences between the index file and the current HEAD commit, paths that have differences between the working tree and the index file, and paths in the working tree that are not tracked by Git.

The first are what you would commit by running git commit; the second and third are what you could commit by running git add before running git commit.

Example

git status -v --verbose

Shows names of files that are changed and the textual changes.

For a list of more options and what they do, have a look at:
https://git-scm.com/docs/git-status

Git Diff

"git diff" is a command to show the changes made in the main file, your commits and the working tree

Git Pull

git pull [] [ […​]]

Updates local with the latest changes from a remote repository into the current branch. It's very important for version control to keep up with the changes and working on the latest updates.

< repository > should be the name of a remote repository. < refspec > can name an arbitrary remote ref (for example, the name of a tag) or even a collection of refs with corresponding remote-tracking branches (e.g., refs/heads/*:refs/remotes/origin/*), but usually it is the name of a branch in the remote repository.
