Explaning the merge flow

The merge flow is a complicated process, maybe not as complicated as we think. You have a main branch for the work and create new branches from that. When you have finished your work and want to merge your work to the main branch you initiate the merge command which has several steps. This guide explanes how you merge a second branch to the main one.

  1. Checkout the main branch and pull (beacuse you should ALWAYS pull)
  2. If you experience conflicts, checkout the note in the end of the text otherwise go to the next step
  3. Write the merge command followed by the branch you want to merge from
  4. You finalize the merge by adding, commiting and pushing the changes
  5. Now you have officially merged to branches!!

NOTE!
If you experience conflicts during your merge into the main branch, you should ALWAYS abort. This is because you should NEVER handle conflicts in the main branch. To merge when encountering conflicts you use the samecommands but you checkout the branch where you did the changes, that is to say you go to the second branch and merge the main into the second branch then handle the conflicts. After the conlflicts are handeled, you checkout the main branch and initiate the merge process once again to merge the second branch into the main one. This is to say you have to do an extra step to merge when encutering conflicts.

GIT MERGE COMAND

  • git checkout "branch name": It put you in the branch
  • git pull: Get you the latest version
  • git merge "branch nanme": It will merge or fusionate from where you are
  • git add . : It will add all your files are linked to the branch
  • git commit -m "write a message here"
  • git push : it will create the merge
  • git merge --abort: It will cancel the merge

CONFLICTS IN THE MERGE

Something to be aware when you do the merge is always to do it from the main to the branch, in that way you will always be secure that if any conflict happens is going to be in the branch.

When you have done the merge it can show some conflic in the code, in that case you should check manually which code you want to keep and to remove everything else. Making sure all the code from up to down is clean. When all has been checked then you can add, commit and push the task.

How to add, commit and push a merged branch to the repository

When you perform a merge, you effectively merge one branch into another—typically a feature branch or bug fix branch into a main branch such as master or develop. Not only will the code changes get merged in, but also all the commits that went into the feature branch

  1. Stash your local working branch changes. Checkout to your local branch
  2. Update your local main branch with remote. Checkout to the main branch
  3. Merge local working branch with main branch.
  4. Get your stash changes and push your working branch.
    5. NOTE! git merge merges the specified branch into the currently active branch.

'git merge --abort'

The “git merge --abort” operation essentially terminates the merger that you have just carried out and separated the two versions of your file,i.e., the current version and the older version.
NOTE! Use git-reset or git merge --abort to cancel a merge that had conflicts. Please note that all the changes will be reset, and this operation cannot be reverted, so make sure to commit or git-stash all your changes before you start a merge.
